# Запуск 2.1 — Пространственное распределение мощности и влияние оптики

## Цель
Подробно исследовать влияние оптики на распределение мощности по матрице сенсора.

Этот запуск должен акцентировать именно оптический этап и отделить
естественную геометрическую неравномерность от шумов и дефектов матрицы.

## Исследовательский вопрос
Насколько сильно закон cos^4 и геометрия оптической системы искажают
равномерность сигнала по полю кадра?

## Постановка
Использовать однородную сцену без пиксельной неравномерности и без дефектных пикселей,
чтобы выделить именно вклад оптики.

Провести прогоны для нескольких температур:
300 K, 340 K, 380 K, 420 K.

Для каждого прогона построить:
- полную карту мощности по матрице
- профили по центральной строке и центральному столбцу
- радиальный профиль ослабления от центра к краю

Дополнительно провести сравнение для нескольких фокусных расстояний или FOV,
если такие параметры можно менять без переписывания модели.

Например:
- узкий FOV
- средний FOV
- широкий FOV

## Что сохранить
В папке results/2.1 сохранить:
- config.json
- optics_profiles.csv
- optics_radial_profile.csv
- arrays/p_distribution_*.csv
- images/p_distribution_heatmap_*.png
- images/profile_row_*.png
- images/profile_col_*.png
- images/radial_profile_*.png
- summary.md

## Какие метрики посчитать
- P_center
- P_edge
- P_corner
- edge_drop_percent
- corner_drop_percent
- radial_nonuniformity_index

Формулы:
- edge_drop_percent = (P_center - P_edge) / P_center * 100%
- corner_drop_percent = (P_center - P_corner) / P_center * 100%

## Что покажет запуск
Запуск покажет, что даже при идеально равномерной сцене сигнал на матрице
не является равномерным из-за геометрии оптики.

Он позволит количественно ответить:
- насколько слабее сигнал на краях;
- как меняется неравномерность при изменении FOV;
- где проходит граница между "оптической нормой" и "аппаратной ошибкой".

## Польза для диплома
Этот запуск особенно полезен для раздела о функциональной схеме и математической модели,
поскольку наглядно связывает:
температуру сцены -> оптический тракт -> пространственное поле сигнала.

Также он помогает обосновать необходимость последующей цифровой коррекции.
