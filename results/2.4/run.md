# Запуск 2.4 — Двухточечная NUC-коррекция и влияние точности коэффициентов

## Цель
Оценить, насколько эффективно двухточечная коррекция неравномерности
уменьшает FPN, и как на качество влияет разрядность коэффициентов.

## Исследовательский вопрос
Как меняется качество коррекции при использовании коэффициентов NUC
с разной фиксированной точностью?

## Постановка
Сначала получить два калибровочных кадра для однородной сцены:
- T1 = 300 K
- T2 = 360 K
или другой диапазон, который не приводит к насыщению АЦП.

По этим кадрам вычислить коэффициенты:
- gain (a)
- offset (b)

Затем применить их к тестовым кадрам:
- 320 K
- 340 K
- 380 K

Для каждой тестовой температуры выполнить коррекцию при нескольких вариантах
дробной точности коэффициентов:
- 3 бита
- 4 бита
- 5 бит
- 6 бит
- full precision

## Какие метрики посчитать
Для каждого тестового кадра и каждой точности:
- std до коррекции
- std после коррекции
- residual_nonuniformity = std_after / std_before
- mean absolute correction error
- center-edge residual
- число заметных кольцевых/градиентных артефактов

## Что покажет запуск
Этот запуск покажет:
- насколько вообще полезна линейная двухточечная коррекция;
- какая разрядность коэффициентов уже достаточна;
- где начинается деградация из-за грубой fixed-point аппроксимации.

## Польза для диплома
Это один из самых сильных запусков для практической части.
Он напрямую связывает математическую модель с инженерной реализацией:
какую точность коэффициентов стоит хранить в памяти, чтобы получить
приемлемое качество коррекции без избыточных аппаратных затрат.
