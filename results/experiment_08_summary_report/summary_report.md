# Итоговый отчет по экспериментам серии 3

Отчет сформирован автоматически на основе сохраненных `summary.json`, `config.json` и CSV-файлов из папок `results/experiment_*/`. Результаты не дописывались вручную.

## Эксперимент 01. Валидация модели формирования ИК-сигнала и метрик SNR/NETD

**Цель.** Проверить базовую радиометрическую зависимость цифрового сигнала от температуры сцены и оценить SNR/NETD.

**Что моделировалось.** Серия однородных ИК-кадров без аномалий при разных температурах сцены и разных реализациях шума. Экспериментальная папка `experiment_01_signal_validation` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** Температура однородной сцены.

**Метрики.** mean_signal_adc, noise_std_adc, snr_for_temp_step, netd_K.

**Главный численный результат.** Средний наклон радиометрической характеристики составил 7.878 ADC/K; лучшая оценка NETD равна 0.239 K.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_01_signal_vs_temperature.png`. Дополнительные PNG в папке: example_frames.png, netd_estimation.png, noise_vs_temperature.png, signal_vs_temperature.png, snr_vs_temperature.png.

**Данные.** CSV-файлы: metrics.csv.

**Вывод для диплома.** Модель дает монотонный рост цифрового сигнала с температурой и позволяет оценивать NETD через наклон характеристики и шум.

## Эксперимент 02. Оценка минимального обнаруживаемого температурного контраста dTmin

**Цель.** Определить минимальный температурный контраст, при котором аномалия начинает стабильно выделяться простым пороговым алгоритмом.

**Что моделировалось.** Кадры с однородным фоном и одной локальной аномалией разной формы при изменяемом температурном контрасте. Экспериментальная папка `experiment_02_min_detectable_contrast` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** Температурный контраст и форма аномалии.

**Метрики.** TPR, FPR, Precision, IoU, detection_probability.

**Главный численный результат.** Минимальный устойчиво обнаруживаемый контраст при IoU>0.3 и вероятности >=0.9: 2 K.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_02_detection_probability_vs_delta_t.png`. Дополнительные PNG в папке: detection_probability_vs_delta_t.png, examples_low_mid_high_delta_t.png, fpr_vs_delta_t.png, iou_vs_delta_t.png, tpr_vs_delta_t.png.

**Данные.** CSV-файлы: metrics.csv, metrics_by_delta_t.csv.

**Вывод для диплома.** При малом dT аномалия маскируется шумом; после порогового контраста вероятность обнаружения резко растет.

## Эксперимент 03. Влияние уровня и типа шума на обнаружение температурной аномалии

**Цель.** Показать, как разные шумовые составляющие ИК-датчика ухудшают качество обнаружения температурной аномалии.

**Что моделировалось.** Одна фиксированная температурная аномалия на фоне при добавлении Gaussian noise, FPN, квантования, дефектных пикселей и комбинированного шума. Экспериментальная папка `experiment_03_noise_influence` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** Тип шума и интенсивность шума.

**Метрики.** SNR-like, TPR, FPR, Precision, IoU.

**Главный численный результат.** Наиболее сильное среднее ухудшение IoU дал шум `combined`: средний IoU=0.497.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_03_iou_vs_noise_level.png`. Дополнительные PNG в папке: example_noise_types.png, iou_vs_noise_level.png, snr_vs_noise_level.png, tpr_fpr_vs_noise_level.png.

**Данные.** CSV-файлы: metrics.csv, metrics_by_noise_level.csv, noise_type_comparison.csv.

**Вывод для диплома.** Рост шума снижает SNR и IoU; разные физические компоненты шума деградируют обнаружение по-разному.

## Эксперимент 04. Оценка эффективности фильтрации ИК-изображений перед обнаружением аномалий

**Цель.** Оценить, улучшают ли простые методы предварительной фильтрации качество сегментации аномалии.

**Что моделировалось.** Одинаковый набор зашумленных ИК-кадров, обработанный различными фильтрами перед применением одного детектора. Экспериментальная папка `experiment_04_filtering` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** Тип фильтра и параметры фильтра.

**Метрики.** SNR before/after, TPR, FPR, Precision, IoU, contrast_change.

**Главный численный результат.** Наиболее устойчивый вариант по среднему IoU: `median_k3` с IoU=0.965 и SNR improvement=10.171.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_04_iou_by_filter.png`. Дополнительные PNG в папке: filter_examples.png, iou_by_filter.png, snr_improvement_by_filter.png, tpr_fpr_by_filter.png.

**Данные.** CSV-файлы: filter_comparison.csv, metrics.csv.

**Вывод для диплома.** Фильтрация снижает шум, но слишком сильное сглаживание может уменьшать локальный контраст аномалии.

## Эксперимент 05. Влияние пространственного разрешения и размера аномалии на обнаружение

**Цель.** Проверить влияние размера аномалии и коэффициента заполнения пикселя на обнаружимость.

**Что моделировалось.** Кадры с прямоугольной аномалией разного пиксельного размера и разного fill_factor при фиксированном температурном контрасте. Экспериментальная папка `experiment_05_spatial_resolution` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** Размер аномалии и fill_factor.

**Метрики.** SNR-like, TPR, FPR, IoU, min_detectable_size.

**Главный численный результат.** Минимальный обнаруживаемый размер при TPR>0.9 и FPR<0.01: 1 px; малый fill_factor заметно ухудшает обнаружение.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_05_tpr_heatmap.png`. Дополнительные PNG в папке: iou_heatmap.png, min_detectable_size.png, size_examples.png, tpr_heatmap.png.

**Данные.** CSV-файлы: metrics.csv, metrics_grid.csv, min_detectable_size.csv.

**Вывод для диплома.** Малые аномалии и неполное заполнение пикселя уменьшают локальный контраст и ухудшают обнаружение.

## Эксперимент 06. Временная инерционность ИК-датчика и усреднение последовательности кадров

**Цель.** Показать влияние временной инерционности датчика и кадрового усреднения на SNR, задержку и качество обнаружения.

**Что моделировалось.** Последовательности кадров со статической, постепенно появляющейся и движущейся аномалией с моделью инерционности первого порядка. Экспериментальная папка `experiment_06_temporal_dynamics` содержит воспроизводимый запуск с параметрами из `config.json`.

**Изменяемые параметры.** alpha фильтра первого порядка и размер окна усреднения.

**Метрики.** SNR-like, TPR, FPR, detection_delay, peak_amplitude_ratio.

**Главный численный результат.** Кадровое усреднение повышало средний SNR до 17.607, но высокая инерционность увеличивала задержку до 2.56 кадров.

**Графики.** Основной график: `/home/tnkqq/Diplom/applause-ir-modelling/results/figures_for_diploma/experiment_06_detection_delay_vs_alpha.png`. Дополнительные PNG в папке: anomaly_amplitude_over_time.png, detection_delay_vs_alpha.png, example_sequence_frames.png, snr_vs_window_size.png, tpr_fpr_vs_alpha.png.

**Данные.** CSV-файлы: amplitude_over_time.csv, metrics.csv.

**Вывод для диплома.** Усреднение полезно для статических объектов, но инерционность ухудшает скорость реакции на появляющиеся и движущиеся аномалии.

## Эксперимент 07. Сравнение алгоритмов обнаружения температурных аномалий на синтетических ИК-кадрах

**Метрики.** Precision, Recall, FPR, F1-score, IoU, AUC, runtime.

**Главный численный результат.** Рекомендуемый базовый алгоритм: `global_threshold` с параметром `k=2.5` (F1=0.872, IoU=0.964).

**Данные.** CSV-файлы: algorithm_summary.csv, metrics.csv.

**Вывод для диплома.** Сравнение показывает компромисс между простотой, устойчивостью к фону, качеством сегментации и временем обработки.
