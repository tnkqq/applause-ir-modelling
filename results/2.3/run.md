# Запуск 2.3 — Дефектные пиксели и деградация качества изображения

## Цель
Исследовать влияние дефектных пикселей на изображение и на статистику выходного сигнала.

## Исследовательский вопрос
Как изменяется качество инфракрасного кадра при росте доли дефектных пикселей,
и насколько эта деградация заметна статистически?

## Постановка
Использовать сцену с однородным фоном и, при возможности, добавить одну локальную
температурную аномалию, чтобы оценить, мешают ли дефекты её обнаружению.

Серия уровней дефектности:
- 0%
- 0.1%
- 0.5%
- 1%
- 2%

Желательно моделировать два типа дефектов:
- "горячие" пиксели (почти максимум)
- "холодные" пиксели (почти минимум)

Для каждого уровня дефектности выполнить 3-5 прогонов с разными seed.

## Что сохранить
В папке results/2.3 сохранить:

- arrays/adc_frame_defect_*.csv
- images/adc_frame_defect_*.png

## Какие метрики посчитать
- число дефектных пикселей
- доля дефектных пикселей
- число выбросов за пределами mean ± 3*std
- mean
- std
- число saturated-like пикселей
- число zero-like пикселей

Если есть аномалия:
- средний сигнал по области аномалии
- контраст аномалии относительно фона
- число дефектов внутри области интереса

## Что покажет запуск
Этот запуск покажет, что дефектные пиксели создают не просто визуальные артефакты,
а могут искажать статистику кадра и ухудшать выделение аномалий.

## Польза для диплома
Полезен для раздела о практических ограничениях системы.
Он позволяет аргументировать необходимость:
- маски дефектов,
- интерполяции неисправных пикселей,
- повторной калибровки при старении сенсора.
