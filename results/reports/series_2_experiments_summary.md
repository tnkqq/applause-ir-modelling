## Run 2.0

Цель: базовый полноразмерный прогон однородной сцены через цепочку `Blackbody -> Optics -> Bolometers -> VectorizedReadout -> ADC`.

Параметры:

- разрешение: `160x120`;
- температуры: `280, 300, 320, 340, 360, 380, 400 K`;
- tolerance: `1e-6`;
- ADC: 10 бит, калиброванное окно `2.238774..111.940680 V`.

Ключевые результаты:

- мощность центрального пикселя выросла от `1.569002e-09 W` до `6.994672e-09 W`;
- средний ADC-код вырос от `75.868` до `901.416`;
- доля saturated-пикселей при `400 K`: `0.000000`;
- доля нулевых пикселей при `280 K`: `0.000000`.

Вывод: базовый конвейер теперь дает рабочий цифровой диапазон и пригодную зависимость `температура -> мощность -> цифровой код`.

## Run 2.1

Цель: выделить вклад оптики в неравномерность поля.

Параметры:

- разрешение: `160x120`;
- температуры: `300, 340, 380, 420 K`;
- FOV cases: narrow, medium, wide.

Ключевой результат:

- максимальное падение мощности в углу: `32.340%` для wide FOV при `300 K`;
- сохранены профили центральной строки, центрального столбца и радиальные профили.

Вывод: даже идеально однородная сцена дает неоднородное распределение мощности из-за геометрии оптического тракта.

## Run 2.2

Цель: оценить вклад разброса параметров микроболометров в fixed pattern noise.

Параметры:

- разрешение: `64x48`;
- температуры: `300, 340, 380 K`;
- tolerances: `1e-6, 3e-6, 1e-5, 3e-5, 1e-4`;
- seeds: `101, 202, 303`;
- оптический градиент отключен для flat-field изоляции FPN.

Ключевой результат:

- максимальное среднее STD по сериям: `120.297 ADC code` при tolerance `1e-4`;
- сохранены зависимости `fpn_std_vs_tolerance.png` и `fpn_range_vs_tolerance.png`.

Вывод: рост tolerance приводит к измеримому росту FPN, что обосновывает необходимость NUC.

## Run 2.3

Цель: оценить влияние hot/cold дефектных пикселей.

Параметры:

- разрешение: `96x72`;
- фон: `300 K`;
- прямоугольная аномалия: `360 K`;
- defect rates: `0%, 0.1%, 0.5%, 1%, 2%`;
- seeds: `11, 22, 33, 44, 55`.

Ключевые результаты:

- при `0%` дефектов средний STD: `118.369`;
- при `2%` дефектов средний STD: `139.637`;
- среднее число выбросов при `2%` дефектов: `229.200`.

Вывод: дефекты ухудшают статистику кадра и могут искажать оценку аномальной области.

## Run 2.4

Цель: проверить двухточечную NUC-коррекцию и влияние разрядности коэффициентов.

Параметры:

- разрешение: `96x72`;
- калибровочные температуры: `300 K` и `360 K`;
- тестовые температуры: `320, 340, 380 K`;
- точности коэффициентов: `3`, `4`, `5`, `6` бит и full precision;
- tolerance: `3e-5`.

Ключевой результат:

- лучший `std_after`: `0.357 ADC code` для `340 K`, full precision;
- при `6 bit` остаточный STD примерно `2.264 ADC code` для `340 K`;
- при `3 bit` остаточный STD примерно `17.903 ADC code` для `340 K`.

Вывод: двухточечная NUC сильно снижает неоднородность; разрядность коэффициентов заметно влияет на остаточный FPN.

## Run 2.5

Цель: связать модель ИК-датчика с прикладной задачей обнаружения температурной аномалии.

Параметры:

- разрешение: `96x72`;
- фон: `300 K`;
- температуры аномалии: `320, 340, 360, 400 K`;
- размеры: small, medium, large;
- tolerance cases: low `1e-6`, medium `3e-5`;
- режимы: uncorrected и corrected.

Для SNR-like введен шумовой пол `1 ADC code`, чтобы идеально выровненный фон после NUC не давал бесконечные значения.

Ключевой результат:

- максимальная SNR-like оценка: `812.680`;
- достигнута для `400 K`, size `large`, tolerance `low`, state `corrected`.

Вывод: эксперимент дает таблицу зависимости видимости аномалии от температуры, размера, tolerance и NUC-коррекции.

## Исправления и инженерные решения

Во время выполнения обнаружены и исправлены проблемы:

- прямое использование старого ADC `vref=3.2 V` приводило к полному насыщению кадров после векторизованного readout;
- вместо этого введено калиброванное ADC-окно для каждой серии;
- в `2.5` добавлен шумовой пол для SNR-like, чтобы избежать деления на численный ноль после почти идеальной NUC-коррекции;
- matplotlib config направлен в `results/.mplconfig`, а этот служебный каталог уже игнорируется git.

