## Run 1.2: Small Full Chain Failed

Папка:

```text
results/1.2/
```

Цель: впервые проверить полный малый конвейер до АЦП.

Пайплайн:

```text
Blackbody -> Optics -> Bolometers -> Readout -> ADC
```

Параметры:

- активное разрешение: `8x6`;
- температуры: `300 K`, `320 K`.

Результат: запуск упал на стадии `Readout.process()`.

Ошибка:

```text
TypeError: only 0-dimensional arrays can be converted to Python scalars
ValueError: setting an array element with a sequence.
```

Причина: `scipy.optimize.fsolve()` возвращает одноэлементный массив, а `Readout.py` присваивает его напрямую в скалярную ячейку массива:

```python
V_int[r,c] = fsolve(...)
```

С современным NumPy 2.x это стало ошибкой. Это важный результат диагностики: код написан под более старое поведение NumPy или требует минимальной правки вида `fsolve(...)[0]`.

Сохраненные файлы:

- `status.json`;
- `summary.md`.

## Главные технические наблюдения

1. Первый участок физической модели (`Blackbody`) работает и дает ожидаемый рост мощности с температурой.
2. `Optics` работает и показывает падение сигнала к краям кадра.
3. Полный конвейер до `ADC` может выполняться на малом разрешении, но зависит от версии NumPy.
4. `Readout.process()` совместим со старым поведением NumPy, но с NumPy 2.x падает на присваивании результата `fsolve`.
5. В `ADC.process()` нет нижнего насыщения в `0`, поэтому при отрицательных напряжениях появляются отрицательные ADC-коды.
6. `Readout.process()` содержит много отладочного вывода, что будет мешать большим экспериментам.
7. Для дипломных запусков лучше начинать с малых разрешений и сохранять все параметры, метрики и stdout.

## Рекомендации для следующего шага

Перед большими экспериментами стоит сделать минимальные технические исправления:

- явно извлекать скаляр из `fsolve(...)[0]` в `Readout.py`;
- добавить нижнее ограничение в `ADC.process()`, если физически цифровой код должен быть неотрицательным;
- убрать или параметризовать отладочный вывод в `Readout.process()`;
- добавить отдельный воспроизводимый сценарий в `experiments/diploma/`, чтобы не запускать эксперименты через одноразовые команды.

Эти изменения лучше выполнять отдельным коммитом после фиксации текущего состояния документации и первых результатов.
